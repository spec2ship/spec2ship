# RT-003: Ready-to-Use Roundtable Templates

**Status**: draft
**Priority**: low
**Category**: Roundtable
**Created**: 2026-01-11
**Updated**: 2026-01-11

## Context

Currently, users must configure roundtables from scratch or use defaults. Pre-configured templates would help users with:
- Common decisions (tech stack selection, architecture patterns)
- Specific domains (API design, security review, UX evaluation)
- Team ceremonies (sprint planning, retrospective)

## Proposal

### 1. Template Structure

```
templates/roundtable/
├── tech-stack-decision.yaml
├── api-design-review.yaml
├── security-assessment.yaml
├── ux-evaluation.yaml
└── architecture-decision.yaml
```

### 2. Template Format

```yaml
name: "Tech Stack Decision"
description: "Evaluate and decide on technology choices"
strategy: "debate"
participants:
  - software-architect
  - technical-lead
  - devops-engineer
agenda:
  - "Current stack assessment"
  - "Alternative evaluation"
  - "Migration considerations"
  - "Final recommendation"
suggested_rounds: 4
```

### 3. Usage

- `/s2s:roundtable --template tech-stack-decision`
- Or interactive selection from template list

### 4. Custom Templates

- Users can create templates in `.s2s/templates/`
- Project-specific templates versioned with code

### Files to Create/Modify
- `templates/roundtable/*.yaml` (new)
- `commands/roundtable/start.md` (add --template flag)
